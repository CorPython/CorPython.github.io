inline function

函数若在class body内定义完成，便自动成为inline function的候选人. class body之外定义的函数，需要加上inline关键字，以此建议编译器将其编译为inline function

constructor—»下面简称ctor

#ifndef _COMPELX_H_
#define _COMPELX_H_

class complex
{
public:
	complex(double r = 0, double i = 0 )
		: re (r), im (i)
	{ }
	//complex(),complex cm,会引起冲突
	/*complex(): re(0), im(0)
	{}*/
	complex& operator += ( const complex& );
	double real () const {return re;}
	double imag () const {return im;}
	void real(double r) { re = r; }
private:
	double re, im;
	friend complex& _doapl(complex* , const complex&);
};

#endif

构造函数中用初值列初始化变量，而最好不在函数体中用"="来赋值 一个变量的构造有两个阶段:初始化,赋值;所以使用初值列的机制效率更好（省去了一个赋值阶段,推荐） ctor可以有多个重载

ctor放在private区

Singleton单例模式

class A
{
public:
    static A& getInstance();
  	setup() {...};
private:
 	A();
  	A(const A& rhs);
  	...
};

A& A::getInstance()
{
 	static A a;
 	return a;
}

通过A::getInstance()来获得对象a,再取调用其他成员函数

const member function

double real() const { return re; }

在函数名的()和{}后加上const

这里的const关键字声明将一定不改变类内的成员变量re

例如：若不写上述的const，以下代码块将会编译错误

const complex c1(1, 2)
cout << c1.real() << c1.imag();

因为使用者创建了一个const的对象c1，即规定c1是一个常量，其中的成员变量也将是const的，没有可能被改变；而在调用成员函数real()时，并未声明其为一个常量成员函数，即类内的成员变量re将"有可能"被改变，这是矛盾的

friend 友元

类中的private成员re与im不能被外界直接取用（可通过类内函数取用）,但类内的友元函数可以直接取得其"朋友"的私有成员

class complex
{
	private:
  	double re, im;
  	friend complex& __doapl(complex*, const complex&);//声明友元函数
};

inline complex& __doapl(complex* ths, const complex& r)
{
 	ths->re += r.re; //自由取得friend的private成员
 	ths->im += r.im;
}

相同class的各个objects互为friends(友元)

{
	complex c1(2,1);
 	complex c2;
 	// c1与c2互为friends
 	c2.func(c1);
}

Pass by value 传值 –> 栈

按值传递将原封不动的复制参数

complex(double r = 0, double i = 0 )
		: re (r), im (i)
{ }

complex r(4,3);

Pass by reference 传递引用

按引用传递相当于传递参数的地址（底层是指针)

complex& operator += (const complex&);

目的是按reference传入一个complex对象的引用,并声明传入的对象本身禁止被函数修改,这就是pass by reference(to const) 参数的传递尽量by reference

Return by value OR Return by reference

return by reference的例子

inline complex& __doapl(complex* ths, const complex& r)
{
  ths->re += r.re;
  ths->im += r.im;
  return *ths;
}
inline complex& complex::operator += (const complex& r)
{
  return __doapl(this, r);
}

注意第一参数传入的是一个指针，指向的对象不是函数体内临时创建的local变量，而是函数外已经存在的某个变量，因而可对其return by reference

什么情况下不能return by reference？ 函数体内的local变量不能return by reference.因为当函数结束，其local变量也消亡了

设计建议

  • 数据放在private中
  • 尽量使用pass by reference
  • 尽量返回值使用reference
  • 类的body中能使用const加上const
  • 构造函数尽量使用Initialize list

特殊操作符重载

有些特殊操作符不能设为成员函数,比如>>和<<

如下,我们重新设置complex类的实部和虚部的数值输出

ostream& operator << (ostream & os,const complex&x)
{
    // return os << "(" << real(x) << "," << imag(x) << ")";
    return os << real(x) << "+" << imag(x) << "i";  
}

注意这时候的os的一直在改变,所以它不是const的引用.

但是为什么这里的返回类型是operator类型呢?因为考虑到有连续输出的需求,比如

cout << c1 << c2;

从这里开始是c++的第二种分类**

Big Three

c++的三大函数分别是: 拷贝构造,拷贝赋值,析构

对于成员中含有指针的一定要含有拷贝构造和拷贝赋值,因为会出现浅拷贝的现象,导致内存泄漏.

所谓浅拷贝就是在进行拷贝只是将指针指向同一块内存来达到内容的一致,但是会导致原先指向的空间的内容变成垃圾,因为这时候没有东西指向它了!

拷贝构造

拷贝构造的参数的类型必须是相同的,注意拷贝构造要分配足够的内存

inline
String::String(const char* cstr)
{
   if (cstr) {
      m_data = new char[strlen(cstr) + 1];
      strcpy(m_data, cstr);
   }
   else {
      m_data = new char[1];
      *m_data = '\0';
   }
}

拷贝赋值

先将左边字符串中的内容清空,再分配一个和右边字符串一样大的空间,最后将右边字符串的内容放到左边的空间中

inline
String& String::operator = (const String& str)
{
    //判断是不是自己赋值给自己,不写结果可能会出错
   if (this == &str)
      return *this;

   delete[] m_data;
   m_data = new char[ strlen(str.m_data) + 1 ];
   strcpy(m_data, str.m_data);
   return *this;
}

if (this == &str)
      return *this;

关于上面这两行的重要性

1.对于自我赋值的效率很高

2.如果没有写这两句,对于浅拷贝的话,如果指向同一块空间,而你又先delete了空间,那么这两个指针相当于无处可指了!会出错!

析构函数

释放掉对象内存

String::~String()
{
   delete[] m_data;
}

ostream& operator<<(ostream& os, const String& str)

os获得的是一个char * 的指针,这样就能得到整个字符串,再输出

内存空间管理

stack 栈

是存在于某个作用域中的一块内存空间.假如当你调用函数时,函数本身会形成一个Stack用来放置它接收的参数,以及返回地址以及local object,在函数中使用的任何变量都是来自于上面的栈.离开作用域自动消亡,栈空间的对象会自动调用析构函数来消亡

heap 堆

是由操作系统提供的一块全局的内存空间,程序可通过new保留字来动态分配.使用完成之后必须手动delete

Static

在当前作用域结束之后仍然存在,直到程序结束

全局对象

一种特殊的static对象,也是直到程序结束消亡

new

先分配内存,再调用ctor

Complex* pc = new Complex(1,2);

分解为一下三步

void* mem = operator new( sizeof(Complex) ); //分配内存,相当于malloc(n)
pc = static_cast<Complex*>(mem); //类型转换
pc->Complex::Complex(1,2); //调用构造函数

delete

先调用dtor,再释放memory

Complex* pc = new Complex(1,2);
delete pc;

分解为两步

Complex::~Complex(pc); // 调用析构函数,相当于free(pc)
operator delete(pc); // 释放内存

array new一定要搭配array delete

string *p = new string[3];
delete[] p;
