递归分解可变参数函数的参数包

#include<iostream>
#include<bitset>

using namespace std;

//当参数到0时,必须定义一个边界,不写这个报错
void print() {}

//...就是pack（包）
template<typename T, typename... Types> //模板参数包
void print(const T&fristArg, const Types &...args) {//types函数参数类型包，args函数参数包
    cout << fristArg << "\t还有参数：" << sizeof...(args) << "" << endl; //用sizeof...(参数包)求得还有参数数量
    print(args...);
}

template<typename... Types>
void print(const Types&...args) {//可并存，谁更泛化调用谁
    //print()
}


int main() {
    print(7.5, "hello", bitset<16>(377), 42);
    print();
    print(15, 16.5, 0101);
    return 0;
}

... 就是一个所谓的pack(包)

  • 用于template parameters，就是template parameters pack（模板参数包）
  • 用于function parameter types，就是function parameter types pack（函数参数类型包）
  • 用于function parameters，就是function parameter pack（函数参数包

一致性初始化–初始化列表 initializer list

c++可以使用{}实现对数组、普通的结构体（没有构造函数和析构函数）的初始化，但是初始化列表并不能对对象和函数使用，因此c++11使用std::initializer_list对这一特性进行了拓展

#include <initializer_list>
#include <iostream>
using namespace std;
class Test {
public:
    Test(initializer_list<int>) {};
};

class Person {
public:
    Person(string _name, int _age, string _id): name(_name), age(_age), id(_id) {};
private:
    string name;
    int age;
    string id;
};


int fun(initializer_list<int> list) {
    return *(list.begin());
};

int main(int argc, char const *argv[])
{
    Test test{1, 2, 3, 4, 5, 6}; //类对象的初始化
    Person p = {"zzz", 15, "0525"}; //类对象的初始化
    cout << fun({1, 2, 3}) << endl;
    return 0;
}

通过{ }来设置初值和之前不用会有哪些需要注意的点呢?

当出现类型窄化时,会出现error,例如在初始化列表时将double给int

int i{};//默认初值为0
int j;//未定义值
int *p;//未定义值
int *q{};//默认为nullptr
//int x{3.2};error: narrowing conversion from 'double' to 'int' inside { }

两种构造函数

C++11提供了两种新的构造函数特性

  • 委托构造(多个构造函数的情况,能够复用其他构造函数代码)
  • 继承构造(类继承)

一个委托构造函数使用它所属类的其他构造函数执行它自己的初始化过程，或者说他把自己的一些（或全部）职责委托给了其他构造函数

#include <iostream>

using namespace std;

class test
{
public:
    int a,b;
    test():a(1){}
    test(int i): b(i){}
};
int main(int argc, char const *argv[])
{
    test t(2);
    return 0;
}

继承构造

c++在继承的时候，需要将构造函数的参数逐个传递到积父类的构造函数中完成父类的构造,如下面的代码,代码很冗余,也占用了很多不必要的空间

struct A
{
    A(int i) {}
    A(double d, int i) {}
    A(float f, int i, const char *c) {}
    //...
};
struct B : A
{
    B(int i): A(i) {}
    B(double d, int i): A(d, i) {}
    B(float f, int i, const char *c): A(f, i, c) {}
    //...
};

因此在c++11中,使用using关键字,通过using 父类名::父类名即可调用父类的所有构造函数

struct A
{
    A(int i) {}
    A(double d, int i) {}
    A(float f, int i, const char *c) {}
    //...
};
struct B : A
{
    using A::A;
};

虚函数重载

由于虚函数的特性，可能会被意外进行重写，为了做到精确对虚函数重载的控制，c++11使用了override和final关键字完成对这一特性的实现.

override关键字

class Test {
public:
    Test() {};
    virtual int fun(int);
};

class Test2 : public Test {
    using Test::Test;
    int fun(int) override;      //显式声明对虚函数进行重载
    //int fun(float) override;    //错误，父类没有这个虚函数
};

final关键字

显式的终结类的继承和虚函数的重载使用

class Test {
public:
    virtual int fun(int) final;
};

class Test2 final: public Test {
    //int fun(int) override;     //非法，因为该虚函数已经设置为final,禁止重载
};

//class Test3 : public Test2 {};  //非法，Test2已经设置为final,禁止作为父类

nullptr和constexpr

nullptr的引入是为了解决NULL的冲突,如下面代码

void func(void *) { }
void func(int) { }

func(NULL);// error: call of overloaded 'func(NULL)' is ambiguous

使用nullptr就可以解决这个问题了

#include <iostream>

using namespace std;

void func(void *) { cout << "func1" << endl; }
void func(int) { cout << "func2" << endl; }

int main(int argc, char const *argv[])
{
    func(nullptr);//output "func1"
    return 0;
}

constexpr变量

C++11规定，可以将变量声明为constexpr，用于编译器在编译阶段来验证变量的值是不是常量表达式。

算数类型、指针、引用都可以定义为constexpr，而IO、自定义类、string等不可以。

constexpr int i = 10;
constexpr int i1 = i+6;
//constexpr string s1 = "123";//错误

constexpr修饰指针时,它的初始化值必须是nullptr or 0 or 一个固定的地址(静态变量,全局变量)。 constexpr修饰的指针是顶层const，而const修饰的指针是底层const

#include <iostream>

using namespace std;

int i = 20;

int main(int argc, char const *argv[])
{
    static int n = 0;
    int m;
    constexpr int *p = 0; //正确
    constexpr int *p1 = nullptr; //正确
    constexpr int *p2 = &i; //正确，i是全局变量，地址固定
    const int j = 10;// 常量
    //constexpr int *p3 = &j;//invalid conversion from 'const int*' to 'int*',constexpr是顶层const
    //constexpr int *p4 = &m; //错误，m是局部变量，地址不固定
    constexpr int *p5 = &n; //正确，n的地址固定
    return 0;
}

explict for ctors taking more than one argument

防止编译器自动调用构造函数将其他数据转换为当前对象

#include <iostream>
#include <algorithm>

using namespace std;

struct Complex
{
    int real, imag;

    Complex(int re, int im = 0)
        :real(re), imag(im)
    {}

    Complex operator +(const Complex& x)
    {
        return Complex((real + x.real), (imag + x.imag));
    }
};
int main(int argc, char const* argv[])
{
    Complex cp(15, 20);
    Complex c = cp + 12;
    cout << cp.imag << " " << cp.real << endl;
    cout << c.imag << " " << c.real << endl;
    return 0;
}

上述利用操作符重载和构造函数,将 12转换为Complex(12,0),再通过操作符重载 +将实部和虚部相加后的对象重新返回

但是如果我们在构造函数使用了explict这个关键字,编译器便不能转换成功

gcc 8.1报错如下

error: no match for 'operator+' (operand types are 'Complex' and 'int')
     Complex c = cp + 12;

这个特性在c++11前也存在,但是只支持有一个不是默认实参的情况,但是现在c++11之后支持存在多个这样的参数.

=default,=delete

=default就是保留编译器实现默认的函数

=delete告诉编译器不要去定义它,所以用于dtor时需谨慎!

当用在构造函数中是,如果没有=default,则在自己定义了构造函数之后编译器不再提供默认的构造函数,

用在拷贝构造,赋值构造和析构函数中,=default,=delete和自己实现的函数只能存在一个,不能会报cannot be overloaded的错误,在其他一般函数中也能使用=delete,用来说明这个函数你不想用了(没啥用),=default默认值为0只适用于虚函数!!

#include <iostream>

using namespace  std;

class Foo
{
public:
    Foo() = default;
    Foo(int i): _i(i){};
    Foo(const Foo& x): _i(x._i) { }
    /*Foo& operator=(const Foo& x)
    {
        _i = x._i;
        return *this;
    }*/
    Foo& operator=(const Foo& x)=default;//cannot be overloaded with 'Foo& Foo::operator=(const Foo&)'
    // Foo& operator=(const Foo& x)=delete;
    //~Foo();
    //~Foo()=default;
    ~Foo()=delete;
private:
    int _i;

};

//void fun()=default;
void func()=delete;
int main(int argc, char const *argv[])
{
    Foo fo();
    return 0;
}

谈Using

谈到using的使用,也就离不开谈到define和typedef,但是后两者不能实现的using可以,比如下一节的Alias Template

head.h

#ifndef _HEAD_H_
#define _HEAD_H_
 
#include <iostream>
using namespace std;
 
class Base{
 
public:
    int value1;
    int value2;
};
 
class Child : private Base{
 
public:
    //重写设定访问权限
    using Base::value1;
};
 
#endif

head.cpp

#include "head.h"

int getSum(const int &a, const int &b) {

    return a + b;
}

//别名
using HelloWorld = int;
typedef int HowOldAreYou;

using SUMFUNCTION = int(*)(const int &a, const int &b);
typedef int (*ADD)(const int &a, const int &b);

int main() {

    HelloWorld value1 = 99;
    HowOldAreYou value2 = 10086;

    cout << "The value1 is : " << value1 << endl;
    cout << "The value2 is : " << value2 << endl;

    Child child;
    child.value1 = 999;
    // 没有访问权限
    // child.value2 = 100; //'int Base::value2' is inaccessible within this context

    ADD p = getSum;
    cout << "The sum Value is : " << p(100, 200) << endl;
    SUMFUNCTION sumFunction = getSum;
    cout << "The sum value is : " << sumFunction(2, 10000) << endl;

    return 0;
}

Alias Template 模板别名

#include <iostream>
using namespace std;

// 定义一个具体类型的别名， using 与 typedef 一样的效果
// 但是模板的别名只能用using（如果typedef也能的话，cpp11就不会加using这个特性了）


// 以下效果一样
using u_flags = std::ios_base::fmtflags;
typedef std::ios_base::fmtflags f_flags;

using u_func = void(*) (int, int);   // 感觉using方式看上去更优雅
typedef void(*f_func) (int, int);

// using 特有
template<typename T>
struct Node { T val; };

template<typename T>
using u_Test = Node<T>;    // ok

//template<typename T>
//typedef Node<T> f_Test;    // error

// 因为 Node<T> 不是一个具体类型， 这样：
typedef Node<int> f_Test;   // ok,  具体类型


// 模板别名
template<class T>
using ptr = T*;


int main()
{
	u_Test<int> t1;
	f_Test t2;

	ptr<int> x;
	int ntmp = 90;
	x = &ntmp;

	cout << *x << endl;
	return 0;

for迭代器

除了原来的begin(),rbegin()和end(),rend()之外,c++11还增加了cbegin(),crbegin()和cend(),crend(),c的意思就是const,迭代器指向的值不允许被改变,将正向迭代器和反向迭代器都加上const

看下面这段代码:

#include<iostream>
#include<vector>

using std::cout;
using std::endl;
using std::vector;

int main()
{
	vector<int> v{ 1,2,3,4,5,6,7,8 };
    
	for (vector<int>::iterator iter = v.begin(); iter != v.end(); iter++)
	{
		cout << *iter << " ";
	}
	cout << endl;
    
	for (vector<int>::const_iterator iter = v.cbegin(); iter != v.cend(); iter++)
	{
		cout << *iter << " ";
	}
	cout << endl;
    
	for (vector<int>::reverse_iterator iter = v.rbegin(); iter != v.rend(); iter++)
	{
		cout << *iter << " ";
	}
	cout << endl;
    
	for (vector<int>::const_reverse_iterator iter = v.rbegin(); iter != v.rend(); iter++)
	{
		cout << *iter << " ";
	}
	cout << endl;
	return 0;
}
